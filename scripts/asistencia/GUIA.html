Reporte de asistencia EIG — cómo funciona

De un archivo plano de Zoom a los informes de la maestría, con un comando.

Python + SQLiteArquitectura hexagonal python-pptxHTML → PDF Sin base de datos en la nube

El flujo en una imagen

participants_*.csv        (archivo plano crudo de Zoom)
      │  ── INTEGRADOR (asistencia-nucleo · hexagonal) ──►
estandar.json             (estándar canónico: persona · sesión · asistencia)
      │  ── schema + loader (SQLite json_each) ──►
reporte_asistencia.db     (base relacional con reglas)
      │  ── consolidados.sql (queries) ──►
métricas                  (% asistencia · riesgo · matriz)
      │  ── generadores ──►
Informe comité (.pptx/.pdf)  +  Informe final (.pdf)
Todo corre local, sin nube. Los umbrales del integrador: PRESENTE ≥ 60 min · PARCIAL ≥ 29 min · si no AUSENTE.

Requisitos (una sola vez)

Necesitas Python 3, SQLite, LibreOffice (para PPTX→PDF), Google Chrome (para HTML→PDF) y poppler (pdftoppm, opcional para screenshots).

bash · macOS (Homebrew) 
brew install sqlite libreoffice poppler
# Chrome ya suele estar instalado en /Applications

Camino rápido — un solo comando

La simulación completa desde el archivo plano de Zoom hasta los informes:

desde la raíz del repo 
bash scripts/asistencia/demo/run_flujo_completo.sh

Genera estandar_desde_plano.json, la base reporte_plano.db y el informe de comité en scripts/asistencia/demo/.

Paso a paso (control total)

1

Extraer: archivo plano → estándar (integrador real)

El integrador vive en src/asistencia_nucleo (vendorizado). Convierte uno o varios CSV de Zoom al estándar canónico.

integrador · nucleo extraer 
PYTHONPATH=src python3 -m asistencia_nucleo.entrypoints.cli extraer \
  --fuente zoom=scripts/asistencia/demo/plano_zoom/participants_sesion1.csv \
  --fuente zoom=scripts/asistencia/demo/plano_zoom/participants_sesion2.csv \
  --programa "Energías Renovables" --cohorte C10 \
  --salida estandar.json
Tipos de fuente: zoom (participants CSV), erc (FORMATO .xlsm con roster), master, matriz. La fuente erc trae la lista completa, así que incluye a los que nunca asistieron.
2

Cargar: estándar → SQLite

El loader usa readfile() + json_each() y lee /tmp/estandar.json.

sqlite3
PYTHONPATH=src python3 -m student_service.entrypoints.cli cargar \
  --estandar estandar.json --db reporte_asistencia.db
3

(Opcional) Densificar ausencias

La fuente Zoom no tiene roster: solo emite filas de quien asistió. Para que las ausencias queden explícitas sobre el roster observado:

sqlite3 · densificar
# o añade --densificar al comando de carga:
PYTHONPATH=src python3 -m student_service.entrypoints.cli cargar \
  --estandar estandar.json --db reporte_asistencia.db --densificar
4

Consultar: los consolidados

Todas las métricas son queries en consolidados.sql (Q1 matriz · Q2 por estudiante · Q3 no asistentes · Q4 por sesión · Q5 maestría).

sqlite3 · ver consolidados
sqlite3 -header -column reporte_asistencia.db \
  ".read scripts/asistencia/consolidados.sql"
5

Generar los informes

Dos entregables, cada uno con su script.

informe para comité (PPTX + PDF)
PYTHONPATH=src python3 -m student_service.entrypoints.cli comite \
  --db reporte_asistencia.db \
  --plantilla "Documentos Referencia/5. Informe para comité curricular Energía de la biomasa ERC10.pptx" \
  --salida informe_comite.pptx
informe final (PDF)
# CSV crudos -> datos del informe
PYTHONPATH=src python3 -m student_service.entrypoints.cli datos-final \
  --encuesta encuesta.csv --docente docente.csv --calificaciones notas.csv \
  --profesor "Nombre Docente" --salida datos.json
# datos -> informe final (HTML + PDF)
PYTHONPATH=src python3 -m student_service.entrypoints.cli final \
  --datos datos.json --salida informe_final
